Хід роботи
1 Встановлення Android Studio
1.1 Завантаження Android Studio
Відвідаємо офіційний вебсайт Android Studio та завантажемо останню версію Android Studio, яка відповідає нашій операційній системі Windows (див. рис. 1).

Рисунок 1 - Завантаження останньої версії Android Studio
1.2 Встановлення Android Studio
Запускаємо інсталяційний файл Android Studio.
Слідуємо покроковим інструкціям майстра встановлення: 
- Погодьтеся з умовами ліцензії.
- Виберіть компоненти для встановлення (Android SDK, Android Virtual Device (AVD) Manager). 
- Вкажіть директорію для встановлення програми та SDK.
Завершуємо встановлення, чекаємо завантаження та налаштування Android SDK.
1.3 Налаштування Android Studio після встановлення
Створюємо новий проект, щоб перевірити, чи працюють усі компоненти належним чином: 
- Виберіть тип проекту (Empty Activity). 
- Встановіть мінімальну версію Android SDK для свого проекту.
1.4 Налаштування емулятора Android
      Запускаємо Android Virtual Device Manager для створення віртуального пристрою Android: 
- Виберіть тип пристрою (телефон, планшет або інший). - Виберіть версію Android для емулятора. 
- Налаштуйте інші параметри (розмір екрану, обсяг пам’яті).
Запускаємо емулятор і перевіряємо його роботу (див. рис. 2).

     
Рисунок 2 - Вигляд емулятора 

2 Створення мінімального Android застосунку
2.1 Створення нового Android проєкту
Відкриємо Android Studio та натиснемо на кнопку "Start a new Android Studio project".
Виберемо шаблон для створення мінімального проєкту (див. рис. 3).
Натиснемо Finish для завершення створення проєкту.

Рисунок 3 - Створення нового Android проєкту
2.2 Ознайомлення зі структурою проєкту
У файлі MainActivity.java/MainActivity.kt знайдемо метод onCreate(), який викликається при створенні Activity. У файлі activity_main.xml переконаємось, що існує базовий елемент (наприклад, TextView).

<TextView
   android:layout_width="wrap_content"
   android:layout_height="wrap_content"
   android:text="Hello World!"
   app:layout_constraintBottom_toBottomOf="parent"
   app:layout_constraintEnd_toEndOf="parent"
   app:layout_constraintStart_toStartOf="parent"
   app:layout_constraintTop_toTopOf="parent" />



2.3 Налаштування емулятора або підключення фізичного пристрою
	Емулятор був створений у пункті 1.4.
2.4 Запуск мінімального застосунку
Натиснемо кнопку Run для компіляції та запуску застосунку. Виберемо емулятор для запуску. Після успішної компіляції застосунок запуститься на вибраному пристрої, де на екрані буде відображено простий інтерфейс із елементом TextView (див. рис. 4).


Рисунок 4 - Інтерфейс із елементом TextView

2.5 Внесення змін до інтерфейсу
Відкриємо файл activity_main.xml у редакторі та додамо кнопку (Button).
Вкажемо властивості для кнопки  у XML-коді: 

<Button  
   android:id="@+id/button"
   android:layout_width="wrap_content"
   android:layout_height="wrap_content"
   android:text="@string/button_name"
   app:layout_constraintBottom_toBottomOf="parent"
   app:layout_constraintEnd_toEndOf="parent"
   app:layout_constraintHorizontal_bias="0.498"
   app:layout_constraintStart_toStartOf="parent"
   app:layout_constraintTop_toTopOf="parent"
   app:layout_constraintVertical_bias="0.409" />

Запустимо застосунок (див. рис. 5).


Рисунок 5 - Вигляд кнопки в емуляторі

2.6 Взаємодія з інтерфейсом
У файлі MainActivity.java додаємо обробку натискання на кнопку: 
- Знайдемо кнопку за її ідентифікатором за допомогою методу findViewById(). 
- Додамо обробник натискання за допомогою методу setOnClickListener().
Змінемо текст елемента TextView при натисканні кнопки (див. рис. 6).

Button button = findViewById(R.id.button);
TextView textView = findViewById(R.id.textview);

button.setOnClickListener(new View.OnClickListener() {
   @Override
   public void onClick(View v) {
       textView.setText("Кнопка натиснута");
   }
});


Рисунок 6 - Обробка натискання на кнопку 
3 Створення GitHub репозиторію та публікація коду застосунку
Перейдемо на GitHub та створемо свій акаунт згідно із вимогами. Створемо новий репозиторій. Відкриємо наш Android-застосунок в Android Studio. Ініціалізуйте Git у нашому проєкті. Перевіряємо, чи всі файли вибрані для публікації, та натискаємо Push.
Після успішного виконання команд перейдемо до нашого GitHub репозиторію в браузері, щоб переконатися, що наш застосунок з'явився в репозиторії, і всі файли доступні (див. рис. 7).
